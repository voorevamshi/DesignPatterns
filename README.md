# DesignPatterns

Design patterns in Java are standard, reusable solutions to recurring software design problems. Categorized into three main types—Creational, Structural, and Behavioral—they provide blueprints that help developers write maintainable and scalable code.

## 1. Creational  Design Patterns
These focus on flexible object creation, decoupling systems from instantiation logic.

-   **[Singleton](notes/creational/singleton.md):**  Ensures a single instance.
-   **[Factory Method](notes/creational/factoryMethod.md) & [Abstract Factory](notes/creational/abstractFactory.md):**  Create objects via methods or families of related objects.
-   **[Builder](notes/creational/builder.md):**  Separates complex object construction.
-   **Prototype:**  Creates objects by cloning.

## 2. Structural Design Patterns

These manage how classes and objects are combined for complex structures.

-   **[Adapter](notes/structural/adapter.md) & Bridge:**  Bridge incompatible interfaces or decouple abstraction from implementation.
-   **Decorator & [Proxy](notes/structura/proxy.md):**  Dynamically add behaviors or control access.
-   **Facade, Composite, & Flyweight:**  Simplify complex subsystems, manage trees, or minimize memory.  

# 3.Behavioral Design Patterns

These manage communication and responsibility between objects.

-   **[Observer](notes/behavioral/observer.md), [Strategy](notes/behavioral/strategy.md), & Command:**  Manage state changes, swap algorithms, or encapsulate requests.
-   **Template Method, State, & Iterator:**  Structure algorithms, handle state changes, or manage collections.
-   **Chain of Responsibility, Mediator, Memento, Visitor, & Interpreter:**  Handle request chains, centralize communication, restore states, add operations, or interpret languages
